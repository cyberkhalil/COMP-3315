Chapter 1: Computer Networks and the Internet

Kurose & Ross, Computer Networking: A Top-Down Approach, 9th edition (2025) – Full lecture coverage

1. What is the Internet?

📡 "Nuts and Bolts" View

  • Billions of connected computing devices: hosts / end systems (PCs, servers, smartphones, IoT).
  • Packet switches: routers & switches forward data chunks (packets).
  • Communication links: fiber, copper, radio, satellite – transmission rate = bandwidth.
  • Network of networks: interconnected ISPs.
  • Protocols control sending/receiving: HTTP, TCP, IP, WiFi, 4G/5G, Ethernet.
  • Standards: RFC (Request for Comments), IETF (Internet Engineering Task Force).
  • "Fun" devices: web‑enabled toaster, smart watches, connected cars, weather forecaster.

🛠️ "Services" View

  • Infrastructure providing services to distributed apps: Web, streaming, email, games, e‑commerce, social media, video conferencing.
  • Programming interface (socket API): allows apps to "connect" and use Internet transport service – analogous to postal service.
  • Provides service options: reliable data delivery, real‑time delivery, etc.
The Internet is a network of networks – access ISPs, regional ISPs, tier-1 ISPs (Level3, AT&T, NTT), and content provider networks (Google, Meta, Microsoft).

2. What is a Protocol?

A protocol defines the format, order of messages exchanged between network entities, and the actions taken upon message transmission/reception. Every communication activity on the Internet is governed by protocols.

Human protocols: “What’s the time?” → reply; introductions.
Network protocols: HTTP request → HTTP response; TCP SYN → SYN-ACK.

3. Network Edge: Hosts, Access Networks & Physical Media

End systems (clients & servers) connect to the edge router via access networks. A host sends packets: it breaks an application message into chunks of L bits and transmits at rate R (transmission delay = L/R).

📡 Cable (HFC)
Hybrid fiber coax, up to 1.2 Gbps downstream, shared medium, FDM.
📞 DSL
Digital Subscriber Line, 24‑52 Mbps down / 3.5‑16 Mbps up, dedicated.
🏠 Home networks
Wireless router + modem (cable/DSL/fiber).
🏢 Enterprise (Ethernet/WiFi)
100 Mbps – 10 Gbps, switches, router.
☁️ Data center networks
10s–100s Gbps links, thousands of servers.
📶 Wireless access
WiFi (802.11) up to Gbps; 4G/5G (100's Mbps over ~10 km); Bluetooth; satellite (Starlink <100 Mbps down, 270 ms geostationary).

🔌 Physical Media

  • Twisted pair (copper): Category 5/6 up to 10 Gbps.
  • Coaxial cable: two concentric conductors, broadband, 100's Mbps per channel.
  • Fiber optic: glass fiber, high speed (10's–100's Gbps), low error, immune to electromagnetic noise.
  • Wireless radio: WiFi, 4G/5G, Bluetooth, microwave, satellite; half‑duplex; affected by reflection, obstruction, interference.

4. Network Core: Packet/Circuit Switching & Internet Structure

📦 Packet switching

  • Store‑and‑forward: entire packet must arrive before forwarding → transmission delay = L/R.
  • Queueing and loss can occur when buffers fill.
  • Efficient for bursty data, no call setup, resources shared.
  • Congestion possible → requires reliable data transfer and congestion control.

🔌 Circuit switching

  • End‑end resources reserved for a "call".
  • FDM (frequency division) or TDM (time slots).
  • Guaranteed performance, but idle resources cannot be used by others (no sharing).
  • Traditional telephone networks.
Example (1 Gb/s link, each user active 10% of time at 100 Mb/s):
• Circuit switching → max 10 users.
• Packet switching → with 35 users, probability that more than 10 are active simultaneously is < 0.0004 → supports many more users.

🌍 Internet structure: a "network of networks"

  • Access ISPs connect to regional / tier‑1 ISPs (Level3, AT&T, NTT).
  • Content provider networks (Google, Meta, Microsoft) build their own networks, bypassing tier‑1 ISPs to bring content closer to users.
  • Two key core functions: forwarding (local action: map arriving packet to output link) and routing (global action: determine source‑destination paths using routing algorithms).

5. Performance: Loss, Delay, Throughput

⏱️ Four sources of packet delay

d_nodal = d_proc + d_queue + d_trans + d_prop

• d_proc: nodal processing (bit error check, output link) – typically < microseconds.
• d_queue: queueing delay – depends on congestion level.
• d_trans = L/R (transmission delay).
• d_prop = d/s (propagation delay; s ≈ 2×10⁸ m/s).

📊 Queueing delay & traffic intensity

Traffic intensity = La/R (a = average packet arrival rate, L = packet length, R = link bandwidth).
• La/R → 0 : small average delay.
• La/R → 1 : delay becomes large.
• La/R > 1 : infinite average delay → buffer overflow and packet loss.

Caravan analogy: 10 cars (packet), toll booth (link), service time = transmission, propagation speed = highway speed. Illustrates store‑and‑forward and the difference between transmission and propagation delay.

📈 Throughput

Throughput = rate (bits/sec) at which bits are transferred from sender to receiver. The bottleneck link determines end‑end throughput = min(Rs, Rc) in the simple two‑link case. In complex networks, the link with the smallest capacity limits overall throughput.

🔎 traceroute

Measures real delays: source sends packets with increasing TTL values; each router returns a message. Example output (UMass to Eurecom):

1  cs-gw (128.119.240.254)  1 ms  1 ms  2 ms
2  border1-rt-fa5-1-0.gw.umass.edu  1 ms  1 ms  2 ms
...
15 fantasia.eurecom.fr  132 ms 128 ms 136 ms

Packet loss: buffers (queues) have finite capacity; a packet arriving at a full buffer is dropped (lost). It may or may not be retransmitted.

6. Network Security

Originally the Internet was designed for "mutually trusting users"; security mechanisms were added later. Attack types:

Packet sniffing
Promiscuous network interface captures all packets (passwords, messages). Defense: encryption.
IP spoofing
Injection of packets with a fake source address. Defense: ingress filtering, authentication.
Denial of Service (DoS)
Overwhelm target with traffic from a botnet. Defense: firewalls, rate limiting.

Lines of defense: authentication (passwords, digital signatures), confidentiality (encryption), integrity checks (MACs, hashes), firewalls (off‑by‑default packet filtering), and IDS/IPS. More in Chapter 8.

Wireshark – a free packet sniffer – is widely used in end‑of‑chapter labs.

7. Protocol Layers, Service Models & Encapsulation

Layering organizes complex systems. Each layer provides services to the layer above using services from the layer below. Changes inside a layer are transparent to other layers.

Internet 5‑layer stack
• Application: HTTP, SMTP, DNS (messages)
• Transport: TCP, UDP (segments)
• Network: IP, routing protocols (datagrams)
• Link: Ethernet, WiFi, PPP (frames)
• Physical: bits on the wire

📦 Encapsulation

At the source: message → add transport header (Ht) → segment → add network header (Hn) → datagram → add link header (Hl) → frame. The destination reverses the process.

message → [Ht|message] → [Hn|[Ht|message]] → [Hl|[Hn|[Ht|message]]]

This modular design enables independent evolution of technologies (e.g., replacing Ethernet with WiFi without affecting upper layers).

8. Internet History: Milestones

  • 1961-1972: Kleinrock (queueing theory), Baran (packet‑switching), ARPAnet (1969), first email, NCP – 15 nodes.
  • 1972-1980: ALOHAnet, Cerf & Kahn internetworking principles (minimalism, best‑effort, stateless), Ethernet (1976), proprietary networks (DECnet, SNA).
  • 1980-1990: TCP/IP deployment (1983), SMTP, DNS (1983), FTP, TCP congestion control (1988) – 100,000 hosts.
  • 1990-2000: Web (HTML, HTTP, Berners‑Lee), Mosaic/Netscape, commercialization, P2P file sharing – 50 million+ hosts, Gbps backbones.
  • 2005-present: Broadband home access (10–100 Mbps), SDN (2008), 4G/5G/WiFi ubiquity, content provider networks, cloud computing (AWS, Azure), ~15 billion connected devices (2023).

9. ISO/OSI Reference Model (7 layers)

The OSI model adds two layers not found in the Internet stack:

  • Presentation layer: encryption, compression, machine‑specific conventions.
  • Session layer: synchronization, checkpointing, recovery of data exchange.

These services, if needed, must be implemented within the application layer in the Internet architecture.

OSI layers: Application → Presentation → Session → Transport → Network → Link → Physical
